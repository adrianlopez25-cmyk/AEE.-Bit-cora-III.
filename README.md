Fundamentos de Seguridad y Auditoría Syslog: Basado en lo que he comprendido, este protocolo nos permite clasificar un evento dependiendo de qué componente del sistema generó el mensaje (denominado Facility) y de otro valor fundamental que define la importancia de dicho evento (Severity). Este último utiliza una escala que va del 0 al 7 donde, cuanto menor es el número, más urgente es la situación (siendo el 0 una emergencia total y el 7 un mensaje de depuración).

Archivo /var/log/auth.log: Este archivo es crítico para la seguridad de un sistema Linux, ya que registra todos los intentos de inicio de sesión. Es fundamental protegerlo porque, en caso de una intrusión, expone vulnerabilidades de enumeración: los atacantes podrían identificar qué nombres de usuario son válidos para centrar sus ataques de fuerza bruta exclusivamente en ellos. Por esta razón, basándose en el principio de menor privilegio, solo deberían tener acceso a esta información los administradores o usuarios con una necesidad justificada.

Errores en conexiones remotas y de usuarios locales: En los registros de errores de conexiones remotas (como SSH), el sistema nos indica el nombre del usuario junto con su dirección IP de origen. Por otro lado, en el caso de los usuarios locales, el log registra el nombre del usuario y las acciones de elevación de privilegios o cambios de identidad que ha intentado realizar dentro del propio sistema.
